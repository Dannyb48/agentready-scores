# AgentReady + DevLake Demo Runbook

## Pre-flight checklist (do before recording)

- [ ] `make run` is running in `devlake/backend/`
- [ ] `yarn start` is running in `devlake/config-ui/`
- [ ] Infrastructure containers up: `podman compose -f docker-compose-dev.yml up -d mysql grafana`
- [ ] `http://localhost:4000` loads without errors
- [ ] `http://localhost:3002` shows Grafana with Fleet Overview, Findings Analysis, Repo Detail dashboards
- [ ] AgentReady connection exists at Connections → AgentReady
- [ ] 6 scopes are registered (konflux-ci/build-service, devlake, release-service, mintmaker, build-definitions, integration-service)
- [ ] Browser tabs open: GitHub repo, DevLake UI, Grafana

---

## Scene 1 — The Problem (~30 sec)

**Show:** `https://github.com/konflux-ci` org page

> "Konflux-CI has dozens of repositories. We want to track AI readiness scores
> across all of them — automatically — and visualize trends over time in DevLake."

---

## Scene 2 — The Central Repository (~1 min)

**Show:** `https://github.com/Dannyb48/agentready-scores`

> "We use a central submissions repository. Each org/repo folder holds
> timestamped assessment JSON files generated by the AgentReady CLI."

1. Click into `submissions/konflux-ci/`
2. Click into any repo folder (e.g. `build-service`)
3. Click the timestamped JSON file — show the score, certification level, findings

---

## Scene 3 — The Runner (~1.5 min)

**Show:** Terminal in `agentready-scores/`

> "This Python runner concurrently assesses all repos and pushes results
> to the central repository."

```bash
# Show the config
cat runner/repos.yaml

# Run it (will re-assess and push)
python runner/assess.py --mode demo --output-dir submissions --workers 3
```

Watch the concurrent `✓` outputs appear. Point out:
- 3 workers running in parallel
- Automatic retry for failures
- Commits and pushes on completion

---

## Scene 4 — DevLake Connection (~1 min)

**Show:** `http://localhost:4000` → Connections → AgentReady

> "The AgentReady plugin is a datasource plugin in DevLake — it has its own
> connection that points to the central submissions repository."

1. Click the existing connection — show the config (repo, path, branch)
2. Click **Manage Scopes** — show 6 repos auto-discovered from the tree
3. Go back to the project — show the blueprint configured

---

## Scene 5 — Run the Pipeline (~1 min)

**Show:** DevLake UI → Project → Blueprint → Run Now

> "Triggering the pipeline pulls all assessment data from GitHub and
> stores it in DevLake."

1. Click **Run Now** (or **Collect Data**)
2. Watch the pipeline tasks execute in real time
3. Show tasks completing: collectSubmissions → extractAssessments → calculateMetrics

---

## Scene 6 — Grafana Dashboards (~2 min)

**Show:** `http://localhost:3002`

### Fleet Overview
> "The Fleet Overview shows aggregate scores across all repos."
- Point out: average score, certification distribution (Platinum/Gold/Silver/Bronze)
- Show the trend over time if multiple runs exist

### Findings Analysis
> "Findings Analysis shows the most common failing checks across the fleet —
> great for prioritizing what to fix org-wide."
- Show the top 15 failures bar chart

### Repository Detail
> "Drill into a single repo for a detailed breakdown."
- Select `konflux-ci/build-service` from the dropdown
- Show score, tier pass rates, per-category breakdown

---

## Scene 7 — The Automation (~30 sec)

**Show:** `.github/workflows/assess-scheduled.yml` in VS Code or GitHub

> "For production, a GitHub Actions workflow runs this automatically every week
> for all public Konflux-CI repositories — no manual steps needed."

Point out:
- Weekly cron schedule
- `workflow_dispatch` for manual triggers
- Concurrent workers configured as input

---

## Tips while recording

- Use **⌘/Ctrl+scroll** to zoom in on important UI elements
- Pause briefly after each scene transition
- If a pipeline run takes too long, have a **pre-completed run** ready to show
- Keep terminal font size at 18px+ for readability
